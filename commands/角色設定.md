# /角色設定 — 常駐角色管理

檢視和修改每個藍圖工作流都會參與的四個常駐角色。

## 四個常駐角色

| 角色 | 代號 | 檔案 |
|------|------|------|
| 組裝師 A | Empath（感性） | `roles/assembler-empath.md` |
| 組裝師 B | Architect（理性） | `roles/assembler-architect.md` |
| 審查官 | Judge | `roles/checker.md` |
| 潤飾師 | Polisher | `roles/polisher.md` |

## 子指令

解析 `$ARGUMENTS` 判斷子指令。沒有指定子指令時，預設執行 `list`。

---

### `/角色設定 list`

1. 掃描 `roles/` 資料夾中所有 `.md` 檔案。
2. 每個檔案讀取前幾行，擷取角色名稱和代號。
3. 以表格呈現：

```
常駐角色：

| 代號 | 角色 | 檔案 | 狀態 |
|------|------|------|------|
| Empath | 組裝師 A | roles/assembler-empath.md | OK / 缺少 |
| Architect | 組裝師 B | roles/assembler-architect.md | OK / 缺少 |
| Judge | 審查官 | roles/checker.md | OK / 缺少 |
| Polisher | 潤飾師 | roles/polisher.md | OK / 缺少 |
```

4. 有角色檔案缺少的話，標記出來並建議建立。

---

### `/角色設定 show <名稱>`

1. 把輸入的名稱對應到檔案。接受以下任一種：代號（`empath`、`architect`、`judge`、`polisher`）、角色名（`assembler-empath`、`assembler-architect`、`checker`）、中文（`感性`、`理性`、`審查`、`潤飾`），或部分匹配。
2. 名稱對應表：
   - `empath` / `感性` / `assembler-empath` / `assembler-a` / `a` -> `roles/assembler-empath.md`
   - `architect` / `理性` / `assembler-architect` / `assembler-b` / `b` -> `roles/assembler-architect.md`
   - `judge` / `審查` / `checker` -> `roles/checker.md`
   - `polisher` / `潤飾` -> `roles/polisher.md`
3. 檔案不存在就回報，並建議建立。
4. 讀取並顯示完整內容。
5. 附上角色職責和邊界的簡短摘要。

---

### `/角色設定 edit <名稱>`

1. 用 `show` 相同的對應表找到檔案。
2. 檔案不存在就問使用者要不要建立。
3. 讀取並顯示目前內容。
4. 問使用者想改什麼。
5. 用 Edit tool 修改。
6. 顯示更新後的檔案。
7. 提醒使用者：修改會影響所有後續的藍圖執行。

---

## 錯誤處理

| 狀況 | 處理方式 |
|------|----------|
| 沒有 roles/ 資料夾 | 建立它並通知使用者 |
| 角色名稱無法辨識 | 顯示名稱對應表，請使用者挑選 |
| 檔案存在但是空的 | 警告並提議用預設模板填入 |
